MBR - находится в первом секторе (т.е. в первых 512Байт)
	- Исполняемый код - отвечает за дальнейшую загрузку (446B)
	- Таблица разделов - (64B) по 16Б на каждый раздел
	- Сигнатура MBR - (2B) Должна быть 55AAh (если нет, то считается что MBR повреждена)


	- В MBR под таблицу раздлелов выделено 64B  по 16B на каждый раздел. Т.е. т.о. в MBR может быть всего 4(первичных/основных) раздела.
		-- При этом дополнительный/расширенный раздел может быть лишь 1.
			-- Т.о. 1 дополнительный и 3 расширенных.
				--- Расширенный раздел(extended partition) - это первичный раздел, который не содержит FS, а содержит другие (логические) разделы.
	- В каждом первичном разделе содержится загрузочный сектор, отвечающий за загрузку ОС с этого раздела.

	С какого именно первичного раздела грузиться - записано в MBR (активный раздел).


GPT - это иной подход к разметки диска.
	В GPT - отпадает потребность в расширенных раздела и как следствие в логических.
		В MBR потребность в них была из за ограничения количества разделов - в GPT таких ограничений нет, по этой причине и потребности в расширенных дисках нет.

Пример создания дисков в Linux:
	1. Получение информации по дискам в момент загрузки (как правило диагностику оборудования начинают с просмотра информации dmesg)
		Информация в ходе работы может быть перезаписана (по этому момент загрузки допонительно сохранятеся в файле /var/log/dmesg. Пример grep sda /var/log/dmesg)

		dmesg | grep disk

	2. Получение информации по подключенным разделам и файловой системе:
		Диски и точки монтирования
			lsblk -fa
		Доступные накопители:
			fdisk -l

	3. Разметка диска (в современных машинах выбриаем GPT с UFI после 2010):
		gdisk /dev/sdb
			начинаем с n - создание раздела (+4G - т.е 4ГБ размер раздела) (или с d - удаление раздела)
			после окончания w - запись.
				Для удаления всех разделов диска:
					x - переход в экспертный режим.
					z - удаление GPT

		Использовалась ранее:
			fdisk /dev/sdb

		А можно и так
			cfdisk /dev/sda

	4. Форматирование разделов
 		mkfs.ext4 /dev/sdb1
 		mkfs -t ext4 /dev/sdb1

 	5. Дефолтное подключение диска задаётся в файле.
 		vim /etc/fstab
 			Дописывается 
 				/dev/sda   /mnt/disk_d  ext4    rw,relatime 0   0

 			Команда для проверки записи в файл fstab
 				findmnt --verify --verbose
