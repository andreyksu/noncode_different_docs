JAVA
	ps ax | grep java -> на выходе получаем <pid>-ява процесса.
	./jstack -l -F <pid> - по сути аналог killall -3 java. Работает долго. Для линуха бесполезно имея kill -3.
	./jmap -F -heap <pid> - показывает состояние памяти jvm. Нужно сравнивать с тем, что отображено в ps. Работает довольно шустро. Данная инфа отображена в kill -3.
	./jmap -F -histo <pid> - показывает число объектов определенного типа и их объем по памяти. Работает долго.
	./jmap -dump:live,format=b,file=heap.bin <pid>
	./jstat -gc -t -h 10 <pid> 100"ms" 30 - показывает состояние/работу gc - нужно при зависании. На сервере Jboss сам логирует работу gc. Число 10 означате кол. строк в группе.
		./jstat -gc -t -h 10 28222 2500"ms" 150

	./jstat -gcutil <pid> <number_samples> <interval_output> - довольно интересная команда, показывает число вызовов GC итд. Возможно интервал и кол. выборок нужно поменять местами.
		./jstat -gcutil 13832 1000 1000
